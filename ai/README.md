# 🎙️ Spartacus Unified AI Middleware (Voice & Email)

[![FastAPI](https://img.shields.io/badge/FastAPI-005571?style=for-the-badge&logo=fastapi)](https://fastapi.tiangolo.com/)
[![Python](https://img.shields.io/badge/python-3670A0?style=for-the-badge&logo=python&logoColor=ffdd54)](https://www.python.org/)
[![OpenAI](https://img.shields.io/badge/OpenAI-412991?style=for-the-badge&logo=openai&logoColor=white)](https://openai.com/)

A highly optimized FastAPI microservice designed to act as the AI intelligence layer for Spartacus Bookings. This unified middleware powers both conversational **Voice Agents** and an automated **Email Beautifier & Responder**, seamlessly integrating with existing backend systems.

## ✨ Key Features

- **🗣️ Voice AI Integration (`/api/voice/booking-lead`)**
  Receives and standardizes booking leads from voice agents. Normalizes dates, times, and activities, preventing duplicate submissions, and forwards them directly to the main backend booking system.
- **🧠 Smart Knowledge Base (RAG) & FAQs (`/api/faq/ask`)**
  A custom multi-layered FAQ answering system. It uses high-priority "Smart Rules" for immediate deterministic answers (like pricing or group sizes), and falls back to semantic search over uploaded PDF knowledge bases for complex queries.
- **✉️ Email AI Automation (`/reply`)**
  Acts as an intelligent email responder. It takes raw backend data or customer questions, retrieves relevant knowledge base context, and dynamically generates professional, formatted email replies via OpenAI.
- **🗄️ Booking State Management**
  Temporarily stores unacknowledged bookings (`/api/bookings`) locally in JSON format, allowing external systems to safely poll and acknowledge (`/api/bookings/{booking_id}/acknowledge`) moving them to an archive.
- **📚 Dynamic Knowledge Rebuilding (`/rebuild-kb`)**
  Allows on-the-fly ingestion and chunking of new PDF documents into the semantic knowledge base index.

## 🚀 Getting Started

### Prerequisites

- Python 3.8+
- An active OpenAI API Key

### Installation

1. **Clone the repository**:
   ```bash
   git clone https://github.com/Rafiuzzaman0028/Joseph_VoiceAssistant_and_Email_Automation.git
   cd Joseph_VoiceAssistant_and_Email_Automation
   ```

2. **Set up a Virtual Environment**:
   ```bash
   python -m venv venv
   # On Windows:
   venv\Scripts\activate
   # On macOS/Linux:
   source venv/bin/activate
   ```

3. **Install Dependencies**:
   ```bash
   pip install -r requirements.txt
   ```

4. **Environment Variables**:
   Create a `.env` file in the root directory and add your credentials:
   ```env
   OPENAI_API_KEY=your_openai_api_key_here
   ```

### Running the Application

Start the FastAPI server using Uvicorn:

```bash
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

The API will be available at `http://localhost:8000`. You can access the interactive Swagger documentation at `http://localhost:8000/docs`.

## 🔌 API Endpoints Overview

- **`GET /health`**: Health check to ensure the AI middleware is running.
- **`POST /api/faq/ask`**: Receives a question payload from the Voice AI and returns an exact or semantic answer.
- **`POST /api/voice/booking-lead`**: Processes new booking requests, deduplicates them, and saves to the local file system while pushing to the backend.
- **`GET /api/bookings`**: Fetches all currently pending (unacknowledged) bookings.
- **`POST /api/bookings/{id}/acknowledge`**: Archives a booking once successfully ingested by the external backend team.
- **`POST /reply`**: Takes an email context payload, queries the KB, and returns a beautifully formatted AI email response.
- **`POST /rebuild-kb`**: Reindexes the PDFs stored in `app/knowledge_base`.

## 🛡️ Architecture & Data Flow

This application acts as a "black box" microservice for the main development team. 
1. **Voice Agents** hit the `/api/faq/ask` for real-time RAG information.
2. **Leads** generated by the agent hit `/api/voice/booking-lead`.
3. The **Main Backend** regularly polls `/api/bookings` to sync these leads into the primary database, then calls `/acknowledge` to clean up the queue.
4. For written communication, the **Main Backend** hits `/reply` to generate context-aware, perfectly formatted emails before dispatching them via SendGrid/AWS SES.
